コツ

まずは受注であれ発注であれプロフィールを充実させる。履歴書のようにかけるものはとにかく書く。ただし実績は実績とみなせるもの。

写真,肩書き,能力など見やすく簡潔にしかし充実させて書く。

またプロフィールを自分で編集したら、それを評価してもらうようにライティングで発注する。例えば私が依頼して納品されたテキストの一部を抜粋する。

肩書きが、独りよがりな印象です。
誰に、何を伝えたいですか？
ご自身が得意なもの、提供できるものをクライアントに一言で表現しましょう。
例:〇〇に豊富な経験がございます。/〇〇が得意です。

複数に登録するのではなく1つに絞る。

発注方法

ライティングの場合について必要なことはまず発注する前にマニュアルを用意すること。

  • 仕様アプリケーション（OSによって使えるアプリが違う）
  • 納品する場合のファイル(txt,doc,docx)
  • 文字コード(UTF-8,ShiftJIS)
  • 全角半角の注意点（数字など）

共通マニュアル

作業はテンプレート.txtという依頼に付属したテキストファイルをダウンロードし、その名前を指定された名前に変更して加筆編集したのち、完成すればそれを納品する。

  • ファイル形式は文字コードがUTF-8のテキストファイルであるようにし、変更しない。
  • テキストファイルを編集するアプリケーションはMicrosoft Wordなどなんでもいいが、ファイルの保存時は納品時に指定された形式でなければならない
  • 全角数字全角英字など半角に変換しうる全角文字は使わない。
  • 漢数字を使うべき箇所があればそれはそのままにする。例えば「一番」は「1番」に変更しない。
  • 日本語の文章の中での句読点は半角記号ではなく日本語で一般的に使われるものにする。「本日は,晴天なり.」はダメでこの場合は「本日は、晴天なり。」にする。

分からないことがあった場合はまずはネットで調べ、それでもわからなかった場合にどのように調べたかを述べたのち尋ねる。

依頼する場合元となるテキストファイルを用意し発注時に添付する。
例えば「ライティングの希望」というテーマで記事作製の依頼があった場合、次のようなテンプレートを配布するので後述のルールに従って編集する。

* 自己紹介
//名前と簡単な自己紹介を書く
//プロフィールを表すためにURLを貼っても良い

* 記事の概要
//あなたが書く記事の概要を書く

* 不満
//現状にある不満を書く

//不満を解決するアイデアを書く

このテンプレートの編集方法は行頭のダブルスラッシュ//のある行はライターへのコメントでありそのコメントどおりの事を書いてから削除する。
この場合「○○を書く」が含まれている行はすべて削除する。ミスがあれば検収は不合格となる。
例えばこのようになる。

ベーシックインカム

選挙とベーシックインカム

選挙時に投票した場合の報償制度を提案する。そして新しい経済と福祉保障の制度としてベーシックインカムというものがある。この発想を弱く借りて組み合わせる。
例えば次の案達のような場合を考える。

案1

ベーシックインカムがあり、1年間での金銭の配布については、以下の条件によって変わるものとする。

  1. 日本の住民であるものにはn万円を配布する
  2. 第一の条件を満たし、かつ、選挙などの投票に1年間においてすべて参加した場合はn+m万円を配布する
  3. 第二の条件に国政選挙が含まれていたならn+m+l万円を配布する

もし仮に地方参政権は外国人に付与されるが国政選挙の参政権は日本国民のみの場合、こうするとn+m+l万円は日本国民のみに配布されることになる。これを弱いBIと呼ぶことにする。

1億人の有権者がいたとしたら支給額n,m,lがそれぞれ1万円なら最大で配布金額は4兆円程度だ。そのうちは購入にあてられ消費税収としてある程度回収される。
また投票権があっても選挙に参加する機会が全く無かった場合は投票したとみなす。

今までは国政選挙の投票時に金銭を配布すること、あるいはすべての選挙に参加すれば金銭の配布を受け取れることを考えていたが、これは投票率を上げるうえであまり効率がよくないかもしれない。
というのも条件を厳しくすると初めから諦める人も出てくるかもしれないからだ。
なので条件を緩め国政選挙でも地方自治体の選挙でも年1回でも何らかの選挙に参加すれば投票奨励金を受け取れるようにすれば投票率は100%近くにならなくても今までより投票率は大きくあがるだろう。
要は投票率を上げることが肝心で100%を求める必要はないということだ。

  • 地方選挙
  • 住民投票

案2　ベーシックインカムとしての国債

案1の代替ではないが配布には国債であてる。
国債は市場で売る場合は除いて基本的に長い期間での利率で現金へと変わる。
ただし、これを個人で購入することは現段階では現実的ではない。なぜなら利率はマイナスになることもありインフレになった場合には金額的には損をする可能性が高いからだ。
金利が負になった場合それはデフレ基調があるからだろうがいずれにせよ個人として購入するにはあまりメリットがない。

そこで消費税収を財源にベーシックインカムとして国債を配布すれば、国債の利率は低くマイナスとなりながら、国民の直接的な財産になる。
流れとしては

  1. 自治体が国からの消費税収からベーシックインカムとして現金を受けとる
  2. 国が一定の国債を発行する
  3. 自治体が国債を買う
  4. 自治体が住人に国債を配布する

となるだろうか。こうすることによって国債は国民と日本に住む住人に身近なものとなり国債に関する知識と関心が増えることが期待できる。

案3　消費税の改革

消費税の税収は2017年度だと17兆円程度だ。まず株式などの有価証券の売買にも消費税を掛けるようにする。

また、名称も購買税と変えた方がいいかもしれない。あるいは、消費税を廃止して証券売買税を導入するというのもいいかもしれない。

しかし問題は軽減税率だ。確かに低所得者向けには助かる面もあるかもしれないが、社会の仕組みが複雑になる。
少子高齢化と人手不足の時代で余計な手間を割さける余力などあるのだろうか。

これらの案は総合的に考え組み合わせることが可能である。最適な経済と政治の関係を構築する為の基礎となる。

投票奨励金 消費税の範囲 ある法人が別の法人の株を買う場合は消費税がかからない。今の日本は法人が貴族として世の中を支配することを良しとする。 これを変え、個人も法人も金融や株を買うときは消費税を払うようにすれば大きな税収増となるだろう。 もちろんそれだけでは景気の悪化は避けられないが弱いベーシックインカムと組み合わされればむしろ景気がよくなる可能性もあるのではないだろうか。 米の配給 前回の記事の続きで国債だけではなく米も配布することを考える。 米を配布するといっても国が米を直接を配布するわけではなく金銭を使い市場を通して弱いベーシックインカムの一種としての配布だ。方法としては 消費税収から国が自治体に金銭を配布する 自治体が米の引換券を住人に配布する 住人が店から引換券で米を購入する 店が自治体から引換券分の金銭を受け取る となるだろうか。 これは米の価格を高く上げ農家の収入や楽にし国全体の農業を安定させる効果があるだろう。 金銭と通貨代替品 国債にしても米にしても通貨の代替と定めたものを受け取る場合は金銭を直接受け取る場合よりも高くなるように設計すれば国の財政と農業と経済を健全化する効果が出るはずだ。例えば金銭であれば月に1万円で代替品なら2万円というぐあいだ。 ただ、金銭か代替品かのどちらか一方の選択は極端なので柔軟なやり方ができれば混乱は少ないだろう。 道州制と特区 日本全国でいきなりするのが難しいなら道州制を導入して一部の特区で実験を行うものおもしろい。北海道と沖縄で先行してみるのはいいかもしれない。なぜならこれら地域は本土から離れている道州のようなものだからだ。そして東京一極集中の弊害を受けている地域でもあるのでそれを解消するためにもいいと確信する。

IT雑務ネットワーク

事務作業におけるIT雑務は何になるだろうか。
まずはパソコンで事務作業を行っている状況を想定しよう。
その場合には当然LANとWANなどのネットワークが構築されているはずだ。もちろんネットワークのない状況でパソコンを使うことも考えられるがそれはITの活用ではない。あくまでITを使うことが主眼であってパソコンそのものに重点を置くものではない。
そうなると重要なのはルーターになる。ルーターがあって初めてLANとWANが活用できる。ではルーターがあってそれを活用できたならどうなるか。 まずはLANのことを考えよう。LANではいくつかの端末機器があり代表的なのはパソコンとプリンターになる。あまり一般的ではないが重要な役割を担うのはNAS(Network Attached Storage)になる。これはファイルサーバー機能を搭載した端末機器でパソコンからファイルの保存機能だけを抜き出してそれに特化しているのようなものだ。
ファイルの共有手段としてはインターネット上のGoogle DriveやMicrosoft OneDriveなどのクラウドストレージもあるが、手続きが必要となる。もちろんこれらも活用すべきであるが、LAN内でどれだけ充実した環境ができるかがまずは重要だ。 そしてWAN、インターネットを活用するには、前述のクラウドストレージ、それに通信とタスク管理が重要になるだろう。通信にはメールが基本になるが2019年現在ではさまざなまサービスがあり何を使用すべきかは一概には言えない。タスク管理であればGoogleカレンダーの使用が一般的だろう。

HTML

HTMLとはHTMLの種類

  • W3C HTML4
  • W3C XHTML1.0
  • W3C XHTML1.1
  • WHATWG HTML5

現在、HTML5 の仕様は、WHATWG と W3C という2つのグループで策定が行われています。

WHATWG と W3C で公開している HTML5 の仕様書は、以下の場所から入手可能です：

  • WHATWG：「HTML Living Standard」
  • W3C：「HTML5 A vocabulary and associated APIs for HTML and XHTML」

HTML5と関連仕様を日本語で読む – 読み解くHTML5 http://html.yomitoku.net/

HTML5が、これまで利用されてきたHTML4とどのような違いがあるのかを、分かりやすく解説したものです。HTML5では、どんな要素・属性がなぜ追加・変更・削除されたかが、一通り説明されています。

HTML5では、要素からプレゼンテーション的な意味を取り除き、さらに、HTML4では定義されていなかった語義を新規の要素として追加しています。プレゼンテーションはCSSに任せ、HTML要素に明確なセマンティクスを与えることで、タグスープを避けようとう考え方がはっきりと表れています。

なお、HTML5そのものがまだ策定中でどんどん変更・改良が行われていますので、このドキュメントも時とともに情報が古くなり、最新のHTML5を正確に表さなくなってしまう点にご注意ください。

W3C - 『HTML 5 differences from HTML 4』日本語訳 - HTML5.JP
