今回は、ウェブサーバーとはなにか？についてみていこうと思う。

まず、WEBサーバーとは、わかりやすく言えばWEBサイト運営には欠かせないものです。

クライアントとと言われるサイトを閲覧する側がダウンロードと言ってサーバーからファイルを受け取りネット上の

を受け取りネット上の画像などを自身のコンピュータに取り込む行為や逆にクライアント側がサーバーに情報

を提供することをアップロードといった行為、具体的には写真の投稿などがそれにあたります。

WEBページをクライアントに表示するのになくてはならない存在と言えるでしょう。

そう考えるとWEBサーバには、大量ともいえるファイルが保存されていることがわかります。

例えば、クライアントが求めている文章の書かれたHTMLファイル、ページ内の構成やデザイン

を調整するCSSファイル、そして、画像ファイルといったものがサーバーという箱には入っています。

ここで注目してもらいたいのは、サーバーという存在です。

彼無では上記したそれぞれのファイルたちは、どうしていいのかわかりません。

彼は、サービスを提供しているソフトウェアやその機能が稼働しているコンピュータ自身を指示しているのですから。

彼の役割としては、先にも述べた通りですが、WEBページ閲覧、メール、ゲーム、ネットそしてSNSといった最近では、

聞かない日がないくらいによく聞くインターネットにかかわるサービスで大活躍なのです。

では次に、サーバーの中身を少し掘り下げていこうと思う。

これだけたくさんの情報をサーバーが持っているだけあってクライアントの要求を即座にかつ正確に取り上げていくため

の工夫された構造があるはずです。

では先に述べたメールに関して見ていきます。メールにはメールサーバーという部門があります。

メールを送受信する際に用います。送信用と受信用でサーバーが分かれています。

それぞれSMTPサーバーとPOPサーバーと言います。別名プロトコルとも言われています。

ここで大事なポイントは、彼らにはポート番号が割り振られているという点です。

たとえるなら、部屋のドアの番号みたいなものです。

SMTPには、ポート５８７番(デフォルト用)でPOPには、ポート１１０番が割り付けられています。

こういったポート番号が、サーバー内のHTTPやHTTPSにも割り付けられています。

こういう構造にしておくことでクライアントからの要求にも即座に答えられますよね。

ちなみに、このポート番号を変更することは、可能ですが何らかの不具合が出ることも予想されるため注意が必要です。

また、このサーバーは極端な話をすると自宅に眠っている古いPCでも利活用が可能。その際にExpressというWEBアプリケ

ーション・フレームワークが必要となってきます。こちらに関しては一連の堅固な機能を提供する

最小限で柔軟なNode.jsと併用できるツールとなっています。
