これから「なぜ？」について解説します。

なぜ忍者AdMaxか

広告は代表的な物はGoogle AdSenseです。しかし、GitHub Pagesで広告を貼ろうとしても審査でなかなか認可されません。
そこで特にそういった制限がない忍者AdMaxを選ぶのがいいでしょう。

なぜFramexsか

そもそも、ページ数が100も行かないような静的サイトの管理にはFramexsが最適です。 静的サイト管理のやりかたとしては、静的サイトジェネレーター（SSG）を使う方法がありますが、変更のたびにサイト生成をしなければなりません。 GitHub Actionsで自動化する方法もありますが、変更後の確認が容易ではありません。 一方、Framexsはクライアント駆動のコンテンツ管理を行うので変更のたびに反映されます。
静的サイトでありながらほぼ動的サイトと同等な雁が可能です。
なのでFramexs一択です。
